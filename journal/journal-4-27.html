Setup

Our project has grown in magnitude; while this has led to questions on our implementation, that is for another discussion. Currently it has been agreed to better test usage on our. While I have done my initial testing. Having a dedicated QA team has its perks and we want to make the best possible use of our talented team in QA. We established 3 stacks. qa1, qa2, qa3.

  1. qa1 - t2.2xlarge -> Control -> Used by QA Manager
  2. qa2 - t2.medium -> Scaling test 2-4 Servers -> Used by QA
  3. qa3 - t2.2xlarge -> Variable -> Used by QA Team

To help deal with issueses presented by QA during load-testing. I created a Cloudwatch Dashboard with metrics from each stack. Crucial ones here being disk usage and ram as we have run out of space multiple times as well as crashed countless times. The dashboard has improved our response time to issues as well as helped us verify our suspicions.

QA has contacted me multiple times regarding the server failure. Upon glancing the dashboard I can quickly deduce what is causing the issue; Rarely have we run out of ram (32 gigs). CPU can spike past 50% and will begin to error if it is maintained past 90% in my assessment (no real suprise). While the cpu metrics are crucial; the star metric in this paragraph is disk space. We've been able to verify disk usage spikes with certain locust test. When the server reaches 100%, its not suprising we are getting failures. 1. Need to create alarm/sns; 2. Cron job; 3. Shared Storage EFS/s3, I believe we store the temp file on the server as well as the processed before sending it to s3. Using Linux to Measure Load

Scaling. I believe scaling is a topic that I continue to learn more and more about. How does scaling physically differ from nginx scaling multiple node applications vs multiple docker containers. It is my assumption that phsical scaling has one undeniable benefit. More cores and more ram. With nginx and docker if we are using the same instance we're effectively splitting resources; I can't imagine this any different from virtualbox where we run ubuntu or kali linux and dedicate resources/ram. This is our max or bottleneck at times. As we will never be able to exceed this level of performance.
